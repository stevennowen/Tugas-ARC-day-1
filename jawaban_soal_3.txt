Fork: Fork adalah tindakan membuat salinan dari repositori Git yang ada di akun pengguna GitHub ke akun pengguna lainnya. Saat Anda melakukan fork pada repositori, Anda membuat salinan independen dari repositori tersebut di akun GitHub Anda sendiri. Fork memungkinkan Anda untuk menyimpan perubahan dan eksperimen pada repositori asli tanpa mempengaruhi repositori utama. Dengan kata lain, Anda dapat membuat perubahan pada salinan fork Anda tanpa memengaruhi repositori asli. Ini sangat berguna dalam kerja tim terbuka atau proyek sumber terbuka di mana kontribusi dari berbagai pengembang diinginkan.

Pull Request: Pull request (PR) adalah permintaan yang diajukan oleh pengembang untuk menggabungkan perubahan yang telah mereka lakukan di cabang mereka di repositori fork ke repositori utama (asli). Jika Anda telah melakukan perubahan pada repositori fork Anda dan ingin mengintegrasikannya ke repositori asli, Anda dapat membuat pull request. Pull request memungkinkan pemilik repositori asli atau kontributor lain untuk meninjau perubahan Anda, memberikan komentar, dan memutuskan apakah perubahan tersebut akan disatukan (merged) ke repositori utama. Ini adalah cara umum untuk berkolaborasi dalam proyek sumber terbuka dan memfasilitasi proses diskusi dan tinjauan kode sebelum perubahan akhirnya digabungkan ke repositori utama.
